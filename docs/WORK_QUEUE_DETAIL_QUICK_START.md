# Work Queue Detail - Quick Start Guide

## How to Access

### Option 1: From My Work Queue Table
In the **My Work Queue** tab, each row now has a blue **Open Detail View** button (external link icon) as the first action button.

Click it to open the full-page detail view for that item.

### Option 2: From Mentions
When you click "View Item" from a mention, if the item is in your queue, it will open in the detail view.

## The Three-Panel Layout

```
┌─────────────────────────────────────────────────────────────┐
│ ← Back to Queue | My Work Queue                    [5 items] │
├──────────┬──────────────────────────┬─────────────────────┤
│          │                          │                     │
│ SIDEBAR  │    DETAILS & ACTIONS     │   COMMUNICATIONS    │
│          │                          │                     │
└──────────┴──────────────────────────┴─────────────────────┘
```

### Left Sidebar - Your Queue
- **Shows:** All items in your work queue
- **Highlighted:** Currently selected item (blue border)
- **Click:** Any item to view its details
- **Badges:** Unread message counts shown in red

### Center Panel - Item Details & Actions
**Top Section - Item Details:**
- Course code and instructor
- Current status (click dropdown to change)
- Priority level
- Title and notes

**Bottom Section - Actions:**
- **Communication:** Staff Message, Message Faculty, Quick Note
- **Workflow:** Mark Complete, Unclaim
- **More:** Dropdown with additional actions

### Right Panel - Communications Thread
- **Faculty Messages** (blue header) - Messages from faculty
- **Staff Messages** (gray header) - Internal staff communications
- **Full Threading** - See replies and conversations
- **Auto-refresh** - Updates when you send messages

## Common Workflows

### Processing an Item
1. Click item in left sidebar
2. Review details in center panel
3. Check communications in right panel
4. Take action (message, change status, etc.)
5. Click **Mark Complete** when done
6. Automatically moves to next item

### Responding to a Message
1. Read message in right panel
2. Click **Reply** button on the message
3. Type your response
4. Submit - appears immediately in thread

### Sending a New Message
**To Staff:**
1. Click **Staff Message** button in center panel
2. Compose message with @mentions if needed
3. Send - appears in Staff Messages section

**To Faculty:**
1. Click **Message Faculty** button
2. Compose message
3. Send - appears in Faculty Messages section

### Quick Notes
1. Click **Quick Note** button
2. Type short note
3. Saves instantly to item

### Changing Status
1. Click status dropdown in center panel
2. Select new status
3. Updates immediately

### Moving Between Items
**Option 1:** Click different item in left sidebar
**Option 2:** Complete current item → auto-moves to next
**Option 3:** Use keyboard (future enhancement)

### Unclaiming an Item
1. Click **Unclaim** button
2. Confirm
3. If it was the selected item, returns to queue

## Tips & Tricks

### Keep Context
The left sidebar always shows your full queue, so you never lose context of what's remaining.

### Unread Indicators
Red badges show items with unread messages - prioritize those first.

### Quick Scanning
Hover over items in the sidebar to see basic info without switching.

### Keyboard Navigation (Future)
We're planning keyboard shortcuts for even faster navigation:
- `j/k` - Next/Previous item
- `c` - Mark complete
- `m` - New message

## When to Use Table vs. Detail View

### Use Table View When:
- You want an overview of all items
- Batch operations (future feature)
- Quick status updates across multiple items
- Comparing multiple items

### Use Detail View When:
- Focused work on individual items
- Reading/responding to communications
- Need more space for details
- Processing items one by one

## Going Back
Click **← Back to Queue** in the header to return to the My Work Queue table view.

Your position in the queue is saved!

## Mobile
On smaller screens, the three panels stack vertically:
1. Sidebar on top
2. Details in middle
3. Communications at bottom

Each section remains independently scrollable.
