## Key Architecture Principles

1. **Separation of Concerns**
   - UI components only handle display
   - Store manages state and business logic
   - Service handles API communication
   - Constants keep magic strings/values centralized

2. **Unidirectional Data Flow**
   - User actions trigger store actions
   - Store updates state
   - Components re-render with new state

3. **Authentication**
   - All API calls include Bearer token
   - Protected routes check authorization
   - Permission-based UI rendering

4. **Error Handling**
   - Try-catch in all async operations
   - User-friendly error messages
   - Error state in store

5. **Code Reusability**
   - Custom hook for common operations
   - Service methods for data transformation
   - Shared components for UI elements

6. **Scalability**
   - Easy to add new filters/sorts
   - Easy to add new API endpoints
   - Easy to extend with new features
